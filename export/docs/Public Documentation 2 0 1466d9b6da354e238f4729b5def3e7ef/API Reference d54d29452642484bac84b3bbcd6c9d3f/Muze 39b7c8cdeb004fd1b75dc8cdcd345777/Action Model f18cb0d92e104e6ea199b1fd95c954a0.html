ActionModel is responsible for registering physical, behavioural actions, side effects

and the mapping between them.

To get access to ActionModel,

const ActionModel = muze.ActionModel;

ActionModel.registerPhysicalBehaviouralMap({..});

for

Syntax: for(...canvases)

Takes an array of canvas instances on which further actions will be performed like registering actions, side effects or enabling interactivity between them.

parameters:
	- canvases:
			required: true
			type: * Canvas
			description: Instance of canvas on which actions will be performed.

* Canvas

Usage

const ActionModel = muze.ActionModel;

// for method attaches the canvases in action model where registration of actions
// and side effects will be done.
ActionModel.for(canvases)
	.registerSideEffects(...sideEffects);

Returns

ActionModel

enableCrossInteractivity

Syntax: enableCrossInteractivity()

By default cross interactivity is disabled between multiple canvases. This method enables cross interaction between the canvases so that any action happening in one canvas gets dispatched on other canvases as well. An optional object can also be passed in this method which is called policy. This policy can be used to control on which canvases the behavioural actions and side effects gets dispatched.

registerPhysicalBehaviouralMap

Syntax: registerPhysicalBehaviouralMap(map)

Registers the mapping of physical and behavioural actions. This mapping is used to establish which behavioural actions should be dispatched on trigger of any physical actions.

parameters:
	- map:
			required: true
			type: object
			description: An object having keys as name of the physical action and value as
an object which specifies the name of the behaviours.

Returns

Instance of action model.

Usage

ActionModel.registerPhysicalBehaviouralMap({
		ctrlClick: {
				behaviours: ['select']
				touch: false // An optional attribute which tells if it is needed to be registered
				// on touch devices or not. By default it is true.
		}
});

Example:

In the above example, we bind ctrlClick physical action with select behaviour and dissociate select behaviour from click action so that only when we ctrl/cmd+click on the element then the select behaviour gets dispatched.

registerBehaviouralActions

Syntax: registerBehaviouralActions(...actions)

Registers behavioural actions on the canvases. It takes definitions of the behavioural actions and registers them on the canvases. Every behavioural action must inherit the GenericBehaviour or VolatileBehaviour or PersistentBehaviour class.

parameters:
	- actions:
			required: true
			type: GenericBehaviour
			description: Behaviour class definition.

Returns

ActionModel

Usage:

In the above example, we register a new behaviour and give it a name singleSelect and map it to click action. Now you can see, the previous data plots are not getting selected and only the current plot is getting selected which is being clicked on.

registerPropagationBehaviourMap

Syntax: registerPropagationBehaviourMap(map)

Registers what behaviour to propagate on dispatch of a certain behavioural action. By default, when any behavioural action is dispatched, then the same behavioural action gets dispatched on all the other canvases which can be changed by using this method. This method is useful when you want to do crossfiltering. Using this method, you can tell muze that a filter behaviour should be dispatched on the other canvases when a select behaviour gets dispatched on the canvas where any physical action has been done. In muze, by default click action is mapped with select behaviour so, on click select behaviour will get dispatched on the canvas which

on propagation to other canvases will dispatch a filter behaviour.

parameters:
	- map:
			required: true
			type: object
			description: Object having key as the source behaviour name and value as the
				propagation behaviour name.
