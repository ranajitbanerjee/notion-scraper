Canvas is a logical component which houses a visualization by taking multiple variable in different encoding channel. Canvas manages lifecycle of many other logical component and exposes one consistent interface for creation of chart. Canvas is intialized from environment with settings from environment and singleton dependencies.

To create an instance of canvas,

const env = Muze();

const canvas = env.canvas();

rows

Gets or sets an array of fields which will be used to create the left and right facets and axis.

When used as a setter,

Syntax: rows(fieldsArr)
parameters
	- fieldsArr:
			required: true
			type: Array
			description: Array of fields required to create the left and right part of
									 the chart.

Returns

Instance of canvas

Usage 

const env = muze();
const canvas = env.canvas();

canvas.rows(['Acceleration']);

Example:

Alternatively, a nested array can also be passed in arguments which will contain two arrays. First array of fields will be used for creating the left facets and axis and the second array will be used for creating the right facets and axis.

Usage

const env = muze();
const canvas = env.canvas();

// Creates a dual axis with horsepower in the left y axis and acceleration in the
// right y-axis.
canvas.rows([['Horsepower'], [ 'Acceleration']]);

Example:

When used as a getter,

Syntax: rows()

color

Gets or sets a field in the canvas which will be used to derive the color of the data plots with respect to the value. Setting color automatically creates a legend in the chart.

When used as a setter,

Syntax: color(colorField)
parameters
	- colorField:
			required: true
			type: string
			description: Name of the field used to derive the color of the data plots.

Usage

const env = muze();
const canvas = env.canvas();

canvas.color('Origin');

Example:

Alternatively, this function can also be passed an object instead of the name of the color field. You can set additional properties like range, step in this object.

Syntax: color(colorConfig)
parameters:
		- colorConfig:
				required: true
				type: Object
				description: Object containing the name of the color field and other 
										 configurations.
				parameters:
						- field:
								required: true
								type: string
								description: Name of the color field.
						- range:
								required: false
								type: Array<string> | string
								description: An array of color strings or name of the color
														 scheme.See all available ColorSchemes* present in
														 muze.
						- step:
								required: false
								type: boolean
								default: false
								description: Creates a step legend.
						- stops:
								required: false
								type: number
								default: 5
								description: Number of stops. There is is only valid when step is
														 true.
						- domain:
								required: false
								type: Array<number>
								description: If specified, it overrides the default color domain
														 which is calculated from the original values in the 
														 datamodel.

Usage

const env = muze();
const canvas = env.canvas();

canvas.color({
	field: 'Acceleration',
	step: true
});

Example:

In the example above, we can see that we have used interpolateGnBu as the color- scheme in the range of color encoding. Almost all the color schemes of d3 under the diverging, sequential and cyclical categories are supported by Muze.

Syntax: size(sizeConfig)
parameters:
		- sizeConfig:
				required: true
				type: object
				description: Size field configuration.
				parameters:
						- field:
								required: true
								type: string
								description:  Name of size field.
						- stops:
								required: false
								type: number
								description: Number of stops determines the number of intervals which
									will be calculated from the size field domain.
						- range:
								required: false
								type: Array<number>
								description: Array of size values.
						- domain:
								required: false
								type: Array<number>
								description: If specified, it overrides the default size domain which is calculated
									from the original values in the datamodel.

detail

Gets or sets an array of fields which increases the level of detail in the chart. For example, if we plot a measure in x-axis vs a measure in y-axis, then only one point will be shown in the chart as there are no dimensional values. Adding dimensions in the detail fields will increase the granularity of the data which will be displayed in the chart whereas adding measures in detail field will not affect the granularity of the data but they will be shown in the tooltip of the chart.

When used as a setter,

Syntax: detail(fieldArr)
parameter:
		- fieldArr:
				required: true
				type: Array
				description: Array of field names.

Returns

Instance of canvas

When used as a getter,

Syntax: detail()

Returns

Array of detail fields present in the canvas.

layers

Gets or sets an array of layer definitions in the canvas which will be used to create various layers in the canvas. If an empty array is passed in arguments, then it disposes the previous layers and renders a default layer based on the type of fields provided in rows and columns.

When used as a setter,

Syntax: layers(layerDefinitions)
parameters:
		- layerDefinitions:
				required: true
				type: Array<LayerDefinition*>
				description: An array of layer definitions

*LayerDefinition

Layer definition is an object which has the following properties. Each layer definition should have

a mark property which is the name of the layer like bar,line,etc. Along with mark each layer definition accepts configurations which are described in details here.

parameters:
		- mark:
				required: true
				type: string
				description: Mark type of layer.
		- encoding:
				required: false
				type: object
				description: Encoding configuration of layer.
		- className:
				required: false
				type: string
				description: Custom class name which will be applied to the dom element of layer.
		- transform:
				required: false
				type: object
				default: {
					type: 'identity'
				}
				description: Transform configuration of layer. Depending on the transform type,
					a grouped or stacked chart is created.
				parameters:
						- type:
								required: false
								type: string
								description: Type of transform.

Returns

Instance of canvas

Layer configurations are described here

Usage

canvas.layers([{
		mark: 'bar',
		encoding: {
				y: 'Acceleration'
		}
}, {
		mark: 'line',
		encoding: {
			y: 'Horsepower'
		}
}]);

When used as a getter,

Syntax: layers()

Returns

An array of layer definitions.

transform

Sets data transformation methods for layers. It takes an object where each key is a unique name which can be referred in the layer definition's source property and the value is a function which takes a datamodel, applies some operations and returns a transformed datamodel. Layers can use the transformed datamodel to create a visualization.

Syntax: transform(transformObj)
parameters:
		- transformObj:
				required: true
				type: object
				description: Data transform configuration.

Returns

Instance of canvas

Usage

const env = muze();
const canvas = env.canvas();

canvas.transform({
	// Create a named data source which can be used by layers.
	averageData: (dm) => dm.groupBy([''])
})
	.layers([{
			mark: 'bar'
	}, {
			mark: 'text',
			// Set the name of the data source defined in transform in source property to
			// be used by this layer.
			source: 'averageData'
	}]);
